# Jyut OCR 詳細校對規則

做 `jyut-ocr` 或相近歷史中文 OCR 專案時用呢份 reference，尤其係全文校對、final pass、長會議報告、合訂講義、表格、票據、印章、尾頁、同常見高信心 OCR 錯字。

如果當前 repo 有 `AGENTS.md`，先跟 repo 本地指示；呢份只係畀 plugin 發佈後，無原 repo 指示嘅人都有一套 portable baseline。

## 並行分工

- 任務有兩份或以上獨立 Markdown/PDF pair，而且工具同用戶授權容許 subagent/delegation 時，預設分派並行 subagent。
- 每個 subagent 只負責一份或一組互不重疊嘅 `NAME.md`/`NAME.pdf`；寫入範圍要明確，唔好多人同時改同一份 Markdown。
- 主 agent 先建立分工清單：檔名、同名 PDF、PDF 頁數或大致頁段、page marker 狀態、表格/尾頁/缺 OCR/跳頁等風險。
- subagent 必須按 PDF 圖像校對，保留原字形同舊式標點，刪頁面家具，處理低信心括號同高信心錯字，最後跑該檔案 residual scan 同 `git diff --check -- TARGET.md`。
- 主 agent 要 review subagent 改動，處理衝突、遺漏同跨檔一致性，並統一做完成前驗證。
- 只有一份 Markdown、同一檔案上下文高度耦合、或 delegation 不可用時，主 agent 直接做；必要時只派 subagent 做只讀疑點、表格、尾頁或頁段審核。

## 原文依據

- 每份 Markdown 優先對同名 PDF，例如 `NAME.md` 對 `NAME.pdf`。
- OCR 係可靠底稿，但唔係最終依據；PDF 圖像先係原文依據。
- `[]` 代表低信心，要優先查；無括號亦可能錯。
- 唔好現代化、正規化、意譯、或者靜靜改順啲字句。
- 保留 PDF 印刷字形，例如 `粤/粵`、`淸/清`、`决/決`、`吿/告`、`塲/場`、`箚/劄/剳/札`、`刧/刼/刦/劫`。
- 保留 PDF 真有嘅舊式標點，包括 `。`、`。。`、`、`。
- PDF 模糊而 OCR 又合理時，保留最有根據嘅讀法或報告待查，唔好憑空補。

## OCR 來源次序

- 缺 OCR 時，立即由 PDF render 圖像做視覺轉寫，唔為任何 OCR 服務停低。
- GJ.cool 古籍 OCR 只係可選背景/並行/極短時間輔助；要已有 `GJCOOL_ACCESS_TOKEN` 或 `GJCOOL_AUTH`，API 正常，未爆 quota。
- 唔等 25 秒或更長；用 5 秒 timeout 睇有冇即時結果。
- GJ.cool 即時返到就用作底稿再核 PDF；timeout、叫等、quota 爆、auth 失敗、服務唔通，就繼續視覺轉寫。
- Apple/macOS OCR、Gemini、Tesseract、OCR JSON 只係提示，除非用戶明確要求。
- 外部研究可以幫手識別人名、官職、地名、日期、典故，但唔可以覆蓋 PDF 字形。

## 頁面家具

只刪明顯非正文：

- 重複頁眉、頁腳、書脊、頁碼、承印資訊、卷冊標籤、藏書印/叢書邊欄、分類文字；
- OCR 工具標題，例如 `gjcool OCR 結果`；
- 空白頁提示，例如 `>此頁OCR無有效文字，可能為空白或損壞掃描。`；
- final output 前刪 `<!-- page_... -->`。

常見家具包括：

- `廣東諮議局第一次臨時會報告書`、`粤東編譯公司承印`、`廣東1議局...`、`廣東諮護局...`、`廣東諮議局薵辦處...`、`議局籌辦處第...`、`諮議局第二次常年會護報吿壹`；
- `廣州大典`、`史部政書類`、輯冊資訊、以及 `經濟學K`、`縱濟學`、`澄 濟` 呢類邊欄課目殘片。

似家具但同正文黐住時，先對 PDF，唔好憑搜尋結果全刪。

## 書目 metadata 同結構

- Markdown 開頭若有 `---` metadata block，要原樣保留喺最前面；唔當正文 heading 或頁面雜訊。
- 新文件只有 PDF 或 OCR 明確提供題名、編者、版本、藏本、影印/編審/校對資料時先補 metadata。
- 全書題名用 `#`。
- 主要議案、報告、呈文、課目、大篇用 `##`。
- `呈文`、`議草`、`批答`、`請議書`、`箚覆`、`劄覆`、章節等用 `###`；再下層按需要用 `####`、`#####`。
- 合訂講義用 `##` 表示主要課目，例如 `法學通論`、`政治學`、`憲法講義`、`行政法學`、`大淸會典要義`、`經濟學`。
- `至...今照修正...` 呢類承接句通常唔係 heading，除非 PDF 版面明顯當題名。
- 目錄完結後如果隨即接正文題名，唔好誤收做上一個 bullet list；要另起 heading。

## 分段同列表

- 校對後同一語義段落合併成一個 Markdown 段落，唔保留 OCR 原始行寬。
- 刪 page comments 後要檢查有冇段落被壓成 soft wrap。
- 上一行結尾係 `第`、`於十`、`未`、`以`、`由`、`可請`、`業經` 等未完語，下一行多半要合併。
- 唔好因兩行都長就機械合併；短行可能係新段、短標題、抬頭敬語。
- 奏摺、呈文、請願書入面 `伏候`、`呈請`、`奉`、`欽奉`、`督部堂`、`旨`、`上諭`、`皇上`、`朝廷` 可能因版面抬頭而獨立。
- 速記錄以發言人轉換作強分段訊號，例如 `某號議員謂`、`督部堂代表謂`、`臬憲代表謂`、`議長謂`。
- 若上一行只係 `某號議員謂`，下一行係同一人發言，PDF 同語法支持時要合併。
- 目錄、條文、問題清單、證據清單、辦法清單可以用 bullet list。
- 保留原文項號，例如 `一`、`二`、`第一條`、`甲一`、`乙一`、`原草一二`、`修正一五`；唔好重新編號。
- 連續 list 入面 item 之間唔加空行。
- item 跨 OCR 行或跨頁延續，要縮入同一 item 底下，唔加空行。
- 縮入行以 `一...`、`甲...`、`乙...` 開頭時，查 PDF 判斷係新 item 定續文。
- bullet 令文義突然斷裂時，考慮係 OCR 將正文誤當 bullet。

## 表格、表單、圖、空白頁

- PDF 明顯係 table，就轉 Markdown table。
- 直排表格要按版面重建欄列，唔好照 OCR 行序。
- 長會議報告嘅金額、人數、地名、罰欵、捐項、拐買男女、縣名清單經常係表格；唔好將數字開頭續行誤作新條。
- 表單、票據、圖要轉錄有意義文字；Markdown table 只喺更忠實時使用。
- table cell 需要多行可以用 `<br>`；final 拉丁掃描命中 `br` 屬預期，但要確認無其他英文字殘留。
- OCR 空白頁可能藏有規則、表格、印章、淡字、尾頁小字或圖；render 後放大檢查先刪。
- 尾頁有印章、黑白反轉、修補塊、淡墨時，要睇普通圖、反色圖、局部 crop。
- 尾頁可能有另一題名或附錄；未能由檔名、封面、目錄、版面確認屬同一 Markdown 前，唔好硬併。
- 最後正文頁如果停喺半句，而下一頁係空白或另一篇，按 PDF 保留半句結尾並報告，唔好自行補完。

## 工作流程

開始前：

- 找同名 PDF/Markdown。
- 確認 PDF 頁數同 render 圖是否完整；如果 pypdf 比現有圖多頁，要補 render 尾頁。
- 查 page comments 是否順序、是否覆蓋最後非空白頁。
- 確認正文由邊頁開始，封面、空白、版權、承印、叢書頁是否應排除。
- 搜 `[]`、`<!-- page_`、`gjcool OCR`、空白 OCR notice、頁面家具、可疑拉丁/數字殘留、高信心錯字模式。

編輯中：

- 按頁或按小段落推進。
- 保留 page comments 作定位；全文審核後先刪。
- 先修低信心括號，再修語境唔通高信心錯字。
- 人名、官職、黨派、公司、地名、日期、案情因果先查 PDF，再視需要查外部資料。
- 按 PDF 版面重建 heading、段落、bullet、table。
- 已校同未校範圍要分清；原始 OCR 後段唔好講成已校對。

完成後：

- 刪所有 page comments 同 OCR notices。
- 重查 heading 層級、list spacing、table spacing、一段一行。
- 搜殘留 brackets、page marker、OCR notice、承印/書脊/叢書資訊、拉丁字母、常見錯字。
- 跑 `git diff --check`。
- 回覆要列明 PDF 頁數、marker continuity、是否逐頁視覺對照、有冇缺 OCR/重建 OCR、疑難字/段、殘留 brackets/page comments/OCR notices、舊標點/合法拉丁詞處理、一段一行檢查、`git diff --check` 結果。

## 高信心 OCR 陷阱

即使無括號，都要按語境同 PDF 查：

- `禁賭/黎賭`、`重大/重火`、`呈文/呈女`、`備文/備女`、`條文/條女`、`查辦/杳辦`、`議决/議次`；
- `綱領/網領`、`總綱/總網`、`聯絡/聯落`、`任期內/任期丙`、`本局/本屈`、`籌/薵`、`膏腴/膏膄`、`盤據/盤拔`；
- `投票/投樂`、`投票法/投樂法`、`監督/監将`、`總會/總命`、`洋藥/洋樂`、`父兄/父兒`；
- `挑釁/挑懋`、`因鬨/因閧`、`釁端/嚴端`、`雇人/雁人`、`輿論/徵視線`、`僅言/第言`、`其間/其問`、`人趨之若鶩/人圖之無之畧識`；
- `法定/已法定`、`中學/巾學`、`提請/捉請`、`發札/發礼`、`札充/礼充`、`膏捐/膏脾`、`三司道/≡司道`、`度支使/度支司`；
- `圖書館/圓書館/岡書館`、`股東/股束`、`股票/股樂`、`官吏/官火`、`大害/火害`、`任令/任合`、`前憲/前前憲`；
- `已/己/巳`、`出/岀`、`聽/聴`、`决/次`、`辦/辨`、`蔓/心`、`費/瞿`、`裹/裏`、`當/常`、`實/寳`；
- `刧/刼/刦/劫`、`箚/劄/剳/札`、`吿/告`、`場/塲`。

## 官職、人名、地名、套語

- `令/合` 常錯：查人名、官名、`法令`、`命令`、`致令`；但 `合就`、`合行`、`合併` 係真官文套語。
- `道/逍`、`遣/逍`、`遺/逍` 常見於 `巡警道`、`北海道`、`隧道`、`差遣`、`遺失`、`載道`。
- `另/男` 會影響議案：查 `另增`、`另籌`、`另案`、`另招商人`；但案情入面 `男女` 要保留。
- `議員`、`代表員`、`督憲代表`、`臬憲代表` 可誤成 `議負`、`化表員`、`督憲化表`、`臬化表`。
- `擔任` 可誤成 `擔作` 或 `抵任`；`年歲` 可誤成 `作歲`；`已成` 可誤成 `以成`；`懸紅` 可誤成 `激紅`；`任其咎` 可誤成 `作其咎`。
- 地名清單要逐字核，例如 `長甯連平長樂南澳` 可能變 `長甯逃平長樂南渙`。
- 人名、地名、官署名唔好靠現代印象直接改，先睇 PDF 字形。
- `一至一`、`云云`、`以下同前議草云云` 可能係原文省文，唔好擅自展開。

## 數字、符號、日期、單位

- `∵`、`≡`、`1` 常代表中文數字或近形字，但絕不可全局替換。
- 查 `第十∵欵`、`其∵曰`、`二≡六五`、`八百九十∵名`、`∵十名` 等。
- `1` 可能係 `壽`、`諮`、`議`、`墨`、`麟`、`窒`、`姑` 或其他形。
- 查 `皇上萬1聖節`、`箚發1詢`、`全體1員`、`應如所1辦理`、`貪1害民`、`建1`、`1司衛門`、`不無1礙`、`未詳報1先電覆`、`陳夔1`。
- 金額單位要按語境：`萬員`、`月費壹員` 多數係 `萬元`、`月費壹元`；但 `議員`、`敎員`、`每員` 係人員。
- 列席、缺席、起立人數、議案次序、人名、罰款、捐項、地名清單要查項號連續性。
- `己/已/巳` 極易混；干支如 `己酉` 要保留。
- 地支次序 `子丑寅卯辰巳午未申酉戌亥` 入面係 `戌`，唔係 `戍`。

## 類型專項

- 議事規則/章程入面，`日程` 可誤成 `曰衣`、`日衣`、`日小`；`議事` 可誤成 `議小`；`審查` 可誤成 `審杳`、`審杏`、`審1`。
- `諮議局章程` 可誤成 `諮議同章程`；`代表員` 可誤成 `化表員`。
- `投票匭`、`投刺匭/投剌匭`、`名刺/名剌` 要逐字核 PDF；真混用要保留，`投聚`、`投票颐`、`投剌軌` 多半係 OCR 錯。
- 長章程內 `第一條`、`第二條` 通常做 bullet item；`附說` 同跨頁續文按版面縮入同一 item。
- 建築/尾頁上下文查 `已成房屋請拆徙者`、`咨請/咨覆`、`庄房/莊房`、`篷廠/蓬廠`、`層板/席枋`。
- 有啲頁會交錯多份短文件；頁頂似 heading 嘅殘片可能係新案，唔一定係上一段續文。

## 拉丁字母同非中文

- final `[A-Za-z]` 掃描命中唔一定係雜訊。
- 法律、政治、經濟講義可能有合法詞源：`Politike`、`Polis`、`jus`、`law`、`droit`、`Recht`、`diritto`、`yu`、`lagu`、`lex`、`directum`。
- 合法拉丁/歐文詞要對 PDF 保留。
- Markdown table 用 `<br>` 時，要接受 `br` 命中，但確認無其他英文字母殘留。

刪 page comments 後查 soft wrap：

```bash
python3 - <<'PY'
from pathlib import Path
lines = Path("TARGET.md").read_text().splitlines()
def structural(s):
    t = s.strip()
    return not t or t.startswith(("#", "- ", "|", ">", "```")) or s.startswith("  ")
print(sum(1 for i in range(len(lines)-1) if not structural(lines[i]) and not structural(lines[i+1])))
PY
```
